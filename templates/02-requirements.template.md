# 요구사항 명세서: RFPilot POC

> 📌 운영 원칙
> - 본 문서는 POC 단계 요구사항만 다룬다. 본 프로젝트 요구사항은 별도 문서로 관리.
> - 우선순위는 POC 성공 기준(`01-poc-plan.md`)과 연결된다.

## 1. 기능 요구사항 (Functional Requirements)

| ID | 기능명 | 설명 | 우선순위 | 검증 시나리오 |
|---|---|---|---|---|
| FR-01 | PDF 문서 업로드 | 사용자가 PDF 파일을 업로드할 수 있다 (최대 10MB) | High | TC-02 |
| FR-02 | 텍스트 추출 | 업로드된 PDF에서 텍스트를 추출한다 (PyMuPDF) | High | TC-02 |
| FR-03 | 벡터 임베딩 | 추출된 텍스트를 청크 단위로 임베딩 후 FAISS 저장 | High | (내부 동작) |
| FR-04 | 질문 입력 | 사용자가 텍스트로 자연어 질문을 입력할 수 있다 | High | TC-01, TC-03 |
| FR-05 | RAG 답변 생성 | 질문에 대해 관련 청크 검색 → LLM으로 답변 생성 | High | TC-03, TC-04 |
| FR-06 | 답변 출력 | LLM이 생성한 답변을 React UI에 표시 | High | TC-01~08 |
| FR-07 | 대화 이력 유지 | 한 세션 내에서 이전 질문 컨텍스트 유지 | Mid | TC-06 |
| FR-08 | 문서 외 질문 처리 | 업로드된 문서와 무관한 질문은 안내 메시지 | Mid | TC-05 |
| FR-09 | 입력 유효성 검사 | 빈 입력, 과도하게 긴 입력 등 안내 처리 | Mid | TC-09, TC-10 |
| FR-10 | 파일 유효성 검사 | PDF가 아닌 파일, 손상된 파일 안내 처리 | Low | TC-11, TC-13 |

## 2. 비기능 요구사항 (Non-Functional Requirements)

| ID | 항목 | 목표값 | 측정 방법 |
|---|---|---|---|
| NFR-01 | 응답 시간 (질문) | 평균 5초 이내 | TC-17 (10회 평균) |
| NFR-02 | 응답 시간 (업로드) | 5MB 기준 10초 이내 | TC-18 |
| NFR-03 | 동시 사용자 | POC 단계 1~5명 | TC-19 |
| NFR-04 | 통신 안정성 | 에러율 5% 이하 | 100회 요청 중 실패 횟수 |
| NFR-05 | 보안 (POC) | 검증만 수행, 차단은 본 프로젝트 | TC-14~16 |
| NFR-06 | 사용성 | 입력→답변까지 클릭 3회 이내 | 사용자 플로우 확인 |
