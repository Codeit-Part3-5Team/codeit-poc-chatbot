## 2. 컴포넌트별 책임

### 2.1 React Frontend
- **역할**: 사용자 입력 받기, 답변 표시
- **주요 화면**:
  - 메인 페이지: PDF 업로드 영역 + 채팅창
  - 업로드 상태 표시 (로딩, 완료, 실패)
  - 채팅 메시지 누적 표시
- **상태 관리**: React `useState` (POC 단계, 라이브러리 없음)

### 2.2 FastAPI Backend
- **역할**: API 엔드포인트 제공, 비즈니스 로직 처리
- **주요 엔드포인트** (자세한 명세는 `08-api-spec.md`):
  - `POST /upload`: PDF 업로드 + 임베딩
  - `POST /chat`: 질문 받아 답변 생성
  - `GET /`: 헬스체크
- **모듈 구조**:
  - `src/server.py`: FastAPI 앱 진입점
  - `src/embedding/`: 임베딩 파이프라인
  - `src/retrieval/`: 벡터 검색
  - `src/generator/`: LLM 호출

### 2.3 FAISS (벡터 DB)
- **역할**: 청크 단위 벡터 저장 + 유사도 검색
- **저장 위치**: 메모리 (POC 단계 — 서버 재시작 시 초기화)
- **본 프로젝트 이월**: Chroma 또는 Pinecone (영구 저장)

### 2.4 OpenAI API
- **임베딩 모델**: `text-embedding-3-small` (한국어 지원, 비용 효율)
- **생성 모델**: `gpt-4o-mini` (응답 속도 + 비용 균형)

### 2.5 세션 저장소
- **POC**: 메모리 dict (`{session_id: [chat_history]}`)
- **본 프로젝트 이월**: Redis (영구 저장, 다중 사용자)

## 3. 데이터 흐름

### 3.1 PDF 업로드 흐름

​```
사용자 PDF 선택
   ↓
[React] FormData로 POST /upload
   ↓
[FastAPI] 파일 검증 (크기 ≤ 10MB, 확장자 .pdf)
   ↓
[FastAPI] PyMuPDF로 텍스트 추출
   ↓
[FastAPI] 텍스트를 1000자 청크로 분할 (overlap 250자)
   ↓
[OpenAI] 청크 임베딩 요청
   ↓
[FAISS] 임베딩 + 원본 청크 저장
   ↓
[FastAPI] {status: "success", chunk_count: 32} 반환
   ↓
[React] "분석 완료" 메시지 표시
​```

### 3.2 질의응답 흐름

​```
사용자 질문 입력
   ↓
[React] POST /chat (session_id, message)
   ↓
[FastAPI] message 검증 (빈 입력, 1000자 초과)
   ↓
[OpenAI] message 임베딩
   ↓
[FAISS] 상위 5개 관련 청크 검색
   ↓
[FastAPI] 검색된 청크 + message + 대화 이력으로 프롬프트 구성
   ↓
[OpenAI] gpt-4o-mini에 프롬프트 전달, 답변 받기
   ↓
[FastAPI] 답변 + session 이력 업데이트
   ↓
[FastAPI] {answer, response_time_ms} 반환
   ↓
[React] 답변 표시 + 대화 누적
​```

## 4. 기술 선택 이유

| 기술 | 선택 이유 | 본 프로젝트 검토 |
|---|---|---|
| **FastAPI** | 비동기 처리로 LLM 호출 효율 ↑, 자동 Swagger 문서화 | 그대로 유지 |
| **React** | 학원에서 다룬 익숙한 스택, Vite로 빠른 개발 | 그대로 유지 |
| **FAISS** | 로컬 인메모리, 설치·운영 간단, POC에 적합 | Chroma 또는 Pinecone |
| **OpenAI gpt-4o-mini** | 응답 속도·비용 균형, 한국어 가능 | 비용 따라 모델 변경 검토 |
| **PyMuPDF** | PDF 텍스트 추출 가장 안정적 | HWP 추가 (pyhwp 또는 hwp5) |

## 5. 보안 고려사항 (POC 단계)

POC 단계에선 **검증만** 수행, 차단은 본 프로젝트에서 구현.

| 항목 | POC 상태 | 본 프로젝트 계획 |
|---|---|---|
| API 키 관리 | `.env` 파일 (Git ignore) | 그대로 + 운영 환경 시크릿 |
| CORS | 로컬 도메인만 허용 | 운영 도메인만 허용 |
| 민감 정보 입력 | 검증만 (TC-14) | 자동 마스킹 |
| 프롬프트 인젝션 | 검증만 (TC-15) | 시스템 프롬프트 강화 + 입력 검증 |

## 6. POC → 본 프로젝트 진화 방향

POC에서 검증되면 본 프로젝트에서 다음과 같이 확장:

​```
[POC: 1대 1 챗봇]              [본 프로젝트: 다중 사용자 시스템]

React (localhost)        →    React (배포)
FastAPI (localhost)      →    FastAPI (Docker)
FAISS (인메모리)          →    Chroma/Pinecone (영구)
세션 (메모리 dict)        →    Redis
PDF만 지원               →    PDF + HWP
보안 검증만              →    보안 차단·마스킹
​```
