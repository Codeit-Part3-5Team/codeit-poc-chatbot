# POC 계획서: RFPilot — RFP 제안서 요약 & QA 상담 챗봇

## 1. 개요

- **프로젝트명**: RFPilot POC (Proof of Concept)
- **기간**: 2026.06.04 ~ 2026.06.18 (2주)
- **PM**: 전민재
- **개발 담당**: 노수민 (서비스 개발 - FastAPI + React)
- **기술 스택**
  - 백엔드: FastAPI (Python 3.11), Uvicorn
  - 프론트엔드: React 18
  - LLM: OpenAI GPT-4o-mini (또는 동급 한국어 모델 후보)
  - 벡터 DB: FAISS (로컬, 운영 단계에서 Chroma 검토)
  - 임베딩: text-embedding-3-small (한국어 지원)
  - 문서 파싱: PyMuPDF (PDF), pyhwp (HWP)

## 2. 목적

본 프로젝트(중급 팀 프로젝트, 2026.06.17~07.08) 진행 전,
다음 두 가지를 검증한다.

1. **기술적 실현 가능성**: PDF/HWP 제안서를 업로드받아
   요약·질의응답하는 RAG 챗봇이 2주 안에 동작 수준으로 만들어지는가?
2. **공수 산정**: 본 프로젝트 3주에 어떤 기능까지 구현 가능한가?
3. **리스크 사전 식별**: 본 프로젝트에서 발목 잡힐 만한 위험 요소를
   미리 발견한다.

## 3. 범위

### ✅ In Scope (POC에서 검증)

- [ ] PDF 제안서 1개 업로드 → 텍스트 추출
- [ ] 추출된 텍스트 → 벡터 DB 저장 (FAISS)
- [ ] 사용자 질문 → 관련 청크 검색 → LLM 답변 생성
- [ ] React UI에서 업로드 + 챗 인터페이스 동작
- [ ] FastAPI ↔ React 통신 안정성
- [ ] 응답 시간 5초 이내 (평균)
- [ ] 샘플 RFP 문서 1~2건으로 시나리오 검증

### ❌ Out of Scope (POC에서 제외)

- HWP 파일 지원 (본 프로젝트로 이월)
- 사용자 인증·회원가입
- 동시 다중 사용자 처리
- 운영 환경 배포 (AWS, Docker 등)
- 다국어 지원
- 모바일 앱
- 민감 정보(주민번호 등) 자동 마스킹 (본 프로젝트 보안 차별화 기능)

## 4. 성공 기준 (Success Criteria)

다음 조건을 **모두** 만족하면 "성공" → 본 프로젝트 진행 결정

| 기준 | 목표값 |
|---|---|
| 시나리오 통과율 | 10개 중 7개 이상 (70%) |
| 평균 응답 시간 | 5초 이내 |
| 통신 에러율 | 5% 이하 |
| 샘플 RFP 요약 품질 | 사람이 읽고 "맥락 맞다" 판단 가능한 수준 |

**부분 성공 기준**: 4개 중 3개 달성 시 일부 수정 후 본 프로젝트 진행
**실패 기준**: 4개 중 2개 이하 달성 시 본 프로젝트 재설계 검토

## 6. 주요 리스크 (초기 식별)

| 리스크 | 영향도 | 발생 가능성 | 대응 방안 |
|---|---|---|---|
| LLM 응답이 5초 이상 걸림 | 높음 | 중간 | 캐싱, 짧은 청크, 타임아웃 3초 |
| 한국어 RFP 문서 임베딩 품질 저하 | 중간 | 높음 | 한국어 특화 임베딩 모델 후보 검토 |

| OpenAI API 비용 누적 | 낮음 | 중간 | 무료 한도 내 테스트, 비용 모니터링 |
